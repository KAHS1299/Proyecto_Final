CRISP-ML Methodology

Tourism Saturation Prediction Process

The project follows CRISP-ML to connect the tourism business problem, the new dataset, model training, evaluation, deployment, and monitoring.

01

Business and Data Understanding

The business goal is to anticipate tourist saturation in Colombian heritage towns so visitors, local authorities, and tourism operators can plan demand peaks before they happen.

Business Objectives

The application predicts whether a destination will have Low, Medium, or High saturation. This supports decisions about travel timing, hotel capacity, mobility planning, staffing, and visitor recommendations.

Visitors

Receive an estimated saturation level, tourist volume, confidence score, and recommendation before planning a trip.

Local Authorities

Use demand indicators to prepare mobility controls, public services, and crowd management actions.

Tourism Operators

Anticipate hotel occupancy, event pressure, and visitor demand by town, month, and season.

Business Questions

  • Which destinations concentrate the highest tourist demand?

    The dashboard compares average historical tourists, estimated tourists, occupancy, capacity, and income by town.

  • Which variables indicate future saturation?

    The model uses town, season, holiday, weather, mobility, events, month, historical tourists, and engineered pressure indicators.

  • What should the user do when demand is high?

    The prediction module returns the saturation level, confidence, estimated tourists, reference occupancy, and a recommendation from the dataset or business rules.

Dataset Used

The active source is data/tourism.csv. It contains 1,980 records for 11 destinations across 2023, 2024, and 2025. The file includes geographic, temporal, climate, mobility, capacity, demand, price, income, recommendation, and saturation variables.

Context Variables
  • town, department, region, lat, lon, altitude_m
  • tourism_type for cultural, coffee, adventure, historical, ecotourism, and religious destinations.
Predictive Variables
  • year, month, season, holiday, school_vacation
  • weather, events, mobility_index, road_traffic, and generated mobility.
Demand and Capacity Variables
  • historical_tourists, estimated_tourists, tourist_capacity, hotel_capacity
  • hotel_occupancy, airbnb_occupancy, saturation_index, tourism_income_usd.
Targets
  • saturation_level is the classification target: Low, Medium, or High.
  • estimated_tourists is used by the regression model to estimate expected visitors.

Technical Viability

Random Forest is appropriate for this project because the dataset mixes categorical and numerical variables, contains non-linear relationships, and needs interpretable feature importance for academic presentation.

Data Leakage Control

The classifier does not use saturation_index or estimated_tourists as input features for classification. The application uses hotel_occupancy as a synchronized reference value from the selected town and month, then generates engineered variables occupancy_pressure and event_pressure.

02

Data Engineering

The dataset is loaded from CSV, column names are normalized, text months are converted to numbers, hotel_occupancy is mapped to occupancy, and mobility is generated from mobility_index.

03

Model Engineering

The project trains two Random Forest pipelines. The classifier predicts saturation_level, and the regressor estimates estimated_tourists. Categorical variables are encoded with OneHotEncoder and numerical variables pass through unchanged.
